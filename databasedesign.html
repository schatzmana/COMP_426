Database Schema

Categories
	CategoryID : INT, primary key
	CategoryName : VARCHAR(30), name of Category e.g.('Furniture', 'Sublease', 'Clothes', 'Textbooks', 'Tutoring', 'Other')

Posts
	PostID : INT, primary key
    Title : VARCHAR(30), title of post
	  Description : VARCHAR(30), description of item being sold
	  Price : INT, price of item being sold
    PhoneNumber : VARCHAR(30), Phone number to contact person selling item
    Picture : VARCHAR(30), url to picture of item being sold
    CategoryID : INT, foreign key to categories table, indicates which category the item is in
    UserID : INT, foreign key to user table, indicates which user the post belongs to

Users
	UserID : INT, primary key
	Username : VARCHAR(30), username of person selling item
    Password : VARCHAR(30), password of person selling item

    
Here is our server-side API: 


    Users:
    POST /users.php
    
    
      
  • Creates new User item from information provided as parameters, which are Username and Password
    • 
      
  • Result is JSON encoding of new User item as
        {"UserID": <integer>,
        "Username":<string>,
        "Password":<string>}
      
    • 
    


    GET /users.php
    
    
      
  • Gets User item using information provided as parameters, which are Username and Password
    • 
      
  • Result is either true or false if Username and Password match with what is in the database
    • 
  


  Posts:
  POST /posts.php
  
    
    
  • Creates new Post item from information provided as parameters, which are Title, Description, Price, PhoneNumber, Picture, and  CategoryID
    • 
    
  • Result is JSON encoding of new Post item as
    {"PostID":<integer>,
    "Title":<string>,
    "Description":<string>,
    "Price":<integer>,
    "PhoneNumber":<string>,
    "Picture":<string>,
    "CategoryID":<integer>}
    
    • 
  


  GET /posts.php
  
    
    
  • Gets Post item using information provided as parameters, which can be either Category or Username
    • 
    
  • Returns JSON representation of Post (see JSON object above) item with Category value matching Category in database or Username matching Username in database
    • 
  


  GET /posts.php/<id>
  
    
    
  • Gets Post item using id passed through
    • 
    
  • Returns JSON representation of Post (see JSON object above) item with the id matching id in database
